📘 프로젝트 개요 문서

프로젝트명

Aone
(부제: 강의 녹음과 자료를 자유롭게 정리하는 AI 학습 파트너)

⸻

1. 프로젝트 개요 (Overview)

Aone은
대학생이 실제 수업에서 사용하는 강의 녹음 파일과 PDF 자료를 기반으로,
AI가 사용자가 구성한 폴더 단위로 자동 정리하고 질문까지 가능한 학습 노트를 만들어주는 웹 서비스이다.

기존 강의 요약 서비스들이
	•	텍스트 기반 요약에 머물러 있거나
	•	단발성 요약에 그치는 한계를 가진 반면,

본 프로젝트는

"강의 녹음 + 강의 자료 + 요약 + 질문"을
사용자가 자유롭게 구성한 폴더 구조로 누적 관리
하는 것을 목표로 한다.

⸻

2. 문제 정의 (Problem)

2.1 실제 대학 강의 환경의 문제
	1.	강의는 대부분 ‘음성’으로 진행됨
	•	학생들은 녹음을 하거나 필기를 병행
	•	하지만 녹음 파일을 제대로 활용할 수 있는 도구가 거의 없음
	2.	PDF 요약 서비스의 한계
	•	텍스트만 요약
	•	그래프/도식/슬라이드 핵심 그림은 해석되지 않음
	•	강의 맥락이 유지되지 않음
	3.	공부 중 질문은 결국 '다시 자료를 뒤지는 과정'
	•	"이거 교수님이 뭐라고 했지?"
	•	"이 개념 어디서 나왔지?"
	•	자료는 있지만 검색과 맥락 연결이 안 됨
	4.	고정된 구조의 불편함
	•	과목/주차로 강제되는 구조는 사용자 스타일에 맞지 않을 수 있음
	•	자유롭게 폴더를 만들고 구성하고 싶은 니즈

⸻

3. 해결 방식 (Solution)

핵심 아이디어

강의 자료를 ‘요약 대상’이 아니라
‘질문 가능한 지식 베이스(RAG)’로 만든다.

핵심 해결 포인트

문제	해결
강의 녹음 활용 불가	🎙 음성 업로드 → STT 자동 전사
PDF 그림 이해 안 됨	🖼 도식 페이지 선별 → AI 캡션
고정된 구조의 불편함	📁 자유로운 폴더 트리 구조 (사용자가 원하는 대로 구성)
질문 시 다시 찾아야 함	🔍 RAG 기반 질문 응답


⸻

4. 서비스 핵심 기능

4.1 폴더 구조 관리
	•	자유롭게 폴더 생성 및 이름 지정
	•	하위 폴더 생성 지원 (폴더 트리 구조)
	•	각 폴더별로 독립적인 자료 관리

4.2 강의 자료 업로드
	•	강의 녹음 파일 (60~90분)
	•	강의 PDF 자료 (20~40p)
	•	원하는 폴더에 직접 업로드

4.3 자동 처리 파이프라인
	•	PDF 업로드 → 슬라이드별 요약 자동 생성
	•	음성 → 텍스트(STT) → 슬라이드와 매칭
	•	슬라이드별 요약 → 전체 요약 생성
	•	벡터 임베딩 저장(RAG)

4.4 요약 시스템
	•	슬라이드별 요약: 각 슬라이드마다 개별 요약 (사용자 수정 가능)
	•	전체 요약: 문서 전체를 종합한 요약 (슬라이드별 요약 + 교수님 음성 녹음본)
	•	폴더 요약: 폴더 내 모든 문서의 요약을 종합한 요약

4.5 질문 & 답변 (RAG)
	•	"이 개념 뭐야?"
	•	"시험 나온다고 한 거 뭐였지?"
	•	특정 폴더 범위로 질문 제한 가능
	•	전체 자료 범위로 질문 가능

👉 모든 답변은 '내 자료'를 근거로 생성

5. 기술적 접근 방식 (Technical Approach)

5.1 전체 아키텍처 개요

Next.js (Frontend + API)
        ↓
Supabase (Auth / Storage / Postgres + pgvector)
        ↓
AI APIs (OpenAI / Gemini)


⸻

5.2 AI 모델 구성 (Best-of-Breed 전략)

기능	사용 모델	이유
음성 인식(STT)	OpenAI gpt-4o-mini-transcribe	분당 과금, 예측 쉬움
임베딩(RAG)	OpenAI text-embedding-3-small	저렴, 안정적
대량 요약	Gemini Flash 계열	긴 텍스트 가성비
PDF 도식 해석	Gemini 멀티모달	이미지 이해 강점
질문 답변(QA)	GPT-4o-mini	설명형 답변 품질

👉 한 모델에 의존하지 않고, 역할별 최적 모델 조합

⸻

6. 데이터 구조 개요

주요 엔티티
	•	Folder (폴더 - 트리 구조 지원)
	•	File (음성/PDF)
	•	Slide Summary (슬라이드별 요약)
	•	Document Full Summary (전체 요약)
	•	Folder Summary (폴더 요약)
	•	Audio Transcript (STT 전사본)
	•	Slide Audio Segment (슬라이드-음성 매칭)
	•	Chunk (RAG용 지식 조각)
	•	Embedding (벡터)
	•	QA Log (질문/답변 기록)

핵심 설계 원칙
	•	파일 본체는 Storage
	•	DB에는 텍스트/메타/벡터만 저장
	•	folder_id를 모든 데이터에 연결 → 사용자가 구성한 구조 유지
	•	폴더는 parent_id로 트리 구조 구현 (루트 폴더부터 하위 폴더까지)

7. 사용자 흐름 (User Flow)
	1.	폴더 생성 및 이름 지정
	2.	필요시 하위 폴더 생성
	3.	강의 녹음 / PDF 업로드 (원하는 폴더에)
	4.	자동 처리(STT / 요약)
	5.	폴더별 요약 확인
	6.	공부 중 질문 (전체 또는 특정 폴더 범위)
	7.	AI 답변 + 출처 확인

	•	시험 정리 시간 ↓
